# Review Capsule Drafts

Guide the user through reviewing locally saved capsule drafts. Drafts are auto-generated by the Stop hook at the end of each session.

## When to Activate

- User runs `/fragcap:review`
- SessionStart hook reports pending drafts
- User asks about their recent explorations

## Flow

1. **Auth gate** — call `check_auth_status`.
   - Not authenticated: "You need to authenticate first. Run `/fragcap:auth`." Then stop.

2. **List drafts** — call `list_local_drafts`.
   - Empty: "No pending drafts. Drafts are auto-generated when you finish a session that produces actionable findings." Then stop.
   - Has drafts: show a numbered summary:

   ```
   Pending drafts (3):

   #1  musicxml-llm-parsing-202604-a7f3
       Problem: Parse sheet music structure with an LLM
       Tags: musicxml, music21, llm-parsing
       Status: resolved | 2 attempts | 1 pitfall | has solution

   #2  pgbouncer-pool-202603-b2e1
       Problem: Connection pool exhaustion under load
       Tags: postgres, pgbouncer, connection-pool
       Status: open | 1 attempt | 0 pitfalls | no solution
   ```

3. **Review each draft** — process one at a time, do not overwhelm the user:

   a. **PII scan** — call `preview_pii(id)`. Display findings if any:

   ```
   PII scan for #1:
   [file_path]     /Users/johnsmith/projects/... -> /Users/username/projects/...
   [email]         john@company.com -> [email]
   [internal_url]  https://api.corp.internal/v1/... -> [internal-url]

   These will be auto-stripped in anonymous mode.
   ```

   If clean: "No PII detected."

   b. **Ask for decision** — three choices per draft:
   - **Push** — ask for visibility:
     - `anonymous` (default): author shown as `gh:anonymous-{hash}`, PII auto-stripped
     - `attributed`: author shown as `gh:{username}`, content as-is
     - Then call `push_capsule(id, visibility)`
   - **Skip** — keep the draft for later, move to next
   - **Delete** — call `delete_draft(id)` to permanently remove

4. **Summary** — after all drafts are processed:

   ```
   Done! Pushed: 2 | Skipped: 1 | Deleted: 0
   ```

## Error Handling

- If `push_capsule` fails: the draft is still saved locally. Tell the user: "Push failed ({error}). Your draft is safe — try again later or check your network."
- If `list_local_drafts` returns an error: suggest checking that the data directory exists.
